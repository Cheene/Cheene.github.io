HTTP首部有哪些？

Jun 22, 2020

14 mins read

H t t p 图解 h t t p
H t t p 图解 h t t p

1 HTTP首部有哪些分类呢？

  • 通用首部字段 General Header Fields
  • 请求首部字段 Request Headers Fields
  • 响应首部字段 Response Header Fields
  • 实体首部字段 Entity Header Fields
  • Cookie 相关字段

2 两大报文首部有哪些

  • 请求报文
    • 报文首部
      • 请求行
      • 请求首部字段
      • 通用首部字段
      • 实体首部字段
      • 其他
    • 空行 CR+LF
    • 报文主体
  • 响应报文
    • 报文首部
      • 状态行
      • 响应首部字段
      • 通用首部字段
      • 实体首部字段
    • 空行 CR+LF
    • 报文主体

3 各个字段的详情

通用首部字段

首部字段名 说明
Cache-Control 控制缓存的行为
Connection 逐跳首部、连接的管理
Date 创建报文的日期时间
Pragma 报文指令
Trailer 报文末端的首部一览
Transfer-Encoding 指定报文主体的传输编码方式
Upgrade 升级为其他协议
Via 代理服务器的相关信息
Warning 错误通知

请求首部字段

首部字段名 说明
Accept 代理可处理的媒体类型
Accept-Charset 优先的字符集
Accept-Encoding 优先的内容编码
Accept-Language 优先的自然语言
Authorization Web认证信息
Expert 期待服务器特定的行为
From 电子邮箱地址
Host 请求的资源服务器
If-Match 比较实体标记
If-Modified-Since 比较资源更新时间
If-None-Match 比较实体标记
if-Range 资源未更新时发送实体Byte的范围请求
If-Unmodified-Since 比较资源的更新时间
Max-Forwards 最大传输逐跳数
Proxy-Authorization 代理服务器要求客户端的认证信息
Range 实体的字节范围请求
Referer 对请求中URI的原始获取方
TE 传输编码的优先级
User-Agent HTTP客户端程序的信息

响应首部字段

首部字段名 说明
Accept-Range 是否接收字节范围请求
Age 推算资源创建经过时间
ETag 资源的匹配的信息
Location 令客户端重定向到指定URI
Proxy-Authenticate 代理服务器对客户端的认证信息
Retry-After 对再次发起的请求的时机要求
Server HTTP服务器的安装信息
Vary 代理服务器缓存的管理信息
WWW-Authenticate 服务器对客户端的认证信息

实体首部字段

首部字段名 说明
Allow 资源可支持的HTTP办法
Content-Encoding 实体主体的可接收编码方法
Content-Language 自然语言
C-Length 大小（字节）
C-Location 代替对应资源的URI
C-MD5 报文摘要
C-Range 位置范围
Type 媒体类型
Expires 过期的日期时间
Last-Modified 最后修改日期

4 缓存代理的分类

  • 端到端首部 End-to-end Header
    • 会转发到最终目标
    • 必须保存在由缓存生成的响应中
    • 必须被转发
  • 逐跳首部 Hop-by-hop Header
    • 首部单词转发有效
    • 可以通过代理而不再转发
    • 需要提供 Connection 首部字段
      • Connection
      • Keep-Alive
      • Proxy-Authenticate
      • Proxy-Authorization
      • Trailer
      • TE
      • Transfer-Encoding
      • Upgrade

5 缓存相关

请求

首部字段名 说明
no-cache 向资源服务器再次验证
no-store 不缓存
max-age=[秒] 响应最大Age
max-stale=[秒] 接收已过期的响应
min-fresh=[秒] 期望时间范围内响应仍有效
no-transform 代理不可更改媒体类型
only-if-cached 从缓存获取资源；无缓存503
cache-extension 新指令标记 Token

响应

指令 说明
public 任意方都可以提供缓存
private 特定用户返回响应
no-cache 不缓存过期的资源
no-store 不缓存请求或响应任何内容
no-transform 代理不可更待媒体类型
must-revalidate 可缓存但必须再向资源服务器进行确认；无法验证 504
proxy-revalidate 中间缓存服务器对缓存的响应有效性再进行确认
max-age=[秒] 响应的最大的Age值
s-maxage=[秒] 公共缓存服务器响应的最大Age值
cache-extension 新指令标记(token)

Connection

  • 控制不再转发给代理的首部字段
  • 管理持久连接
    • Connection: Close 关闭
    • Connection: Keep-Alive 保持连接

Date 创建HTTP报文的日期和时间

Pragma 一般用于告诉中间件不需要缓存 Pragma:no-cache

Trailer 此时在报文主体的后面仍然会有首部字段，Trailer 中表示还有哪一些字段在后面

Transfer-Encoding 规定传输报文的编码方式

Upgrade 检测是否可以使用更高的版本进行通信。参数可以指定不同的通信协议

HTTP/1.1 101 Switching Protocols
Upgrade: TLS/1.0, HTTP/1.1
Connection: Upgrade

Via

常应用于客户端和服务器之间的请求和响应报文的路径

每经过一个代理会添加该代理的标志

Via 经常和 TRACE 方法一起使用。

Warning

Warning:[警告码][警告的主机:端口号][警告的内容](日期时间)
警告码 内容 说明
110 Response is stale 代理返回过期资源
111 Revalidation failed 代理再验证失败
112 Disconnection operation 代理与互联网连接故意切断
113 Heuristic expiration 响应的使用期超过24小时
199 Miscellaneous warning 任意的警告内容
214 Transformation applied 代理处理过编码或媒体类型
299 Miscellaneous persistent warning 任意的警告内容

6 请求首部字段

  • Accept 确定媒体接收的范围

  • Accept:text/html,application/xhtml+html
    • 媒体类型
    • 文本文件
    • 图片文件
    • 视频文件
    • 应用程序使用的二进制文件

  • Accept-Charset 字符集

  • Accept-Encoding 编码格式

    • gzip
    • compress
    • deflate
    • identity

  • Accept-Language 可接受的自然语音

    Accept-Language: zh-cn,zh;q=0.7,en-us,en;q=0.3

  • Authorization 认证信息

  • Expect 客户端告知服务端希望发生什么样的结果

    • 417 Expectation Failed

  • From 告诉服务器，用户代理的钉子邮箱地址

  • Host

    • 告知服务端的主机名和端口号
    • 必须在请求的首部字段包含
    • 主要是因为，单台WEB服务器运行多台主机时需要

  • If-match 携带条件的请求

  • IF-Modified-Since

    If-Modified-Since: Thu, 13 Apr 2020 00:00:00 GMT

    在 Thu, 13 Apr 2020 00:00:00 以后秒如果资源有更新便返回

  • If-Range

    • 条件符合时，返回部分信息

      206 partial Content
content-Range: bytes 20001-10000/10000
content-Length: 5000

    • 条件不符合时，返回全部信息

      200 OK
ETag: "56790"

  • Max-Forwards

    • 每转发一次减一
    • 数值为0的时候返回

  • proxy-Authorization 客户端与代理之间的认证方式

  • Referer 服务器查看当前请求的URI是从哪个WEB页面发去的

  • TE 客户端告知服务器能够处理的传输编码的方式

  • User-Agent 客户端的浏览器类型

7 响应首部字段

  • Accept-Ranges : bytes 告知客户端能否处理范围请求

    • bytes 可以
    • none 不可以

  • Age 客户端的这次请求再多久之前请求过，单位以秒计算

  • ETag 告知客户端实体标识

    • 强 ETag 微小的改变也会改变其值

      ETag: "usagi-1234"

    • 弱 ETag 只是用来提示资源是否相同，除非存在根本性的变化，

      ETag:W/"usagi-1234"

  • Location 通知客户端选择新的地址访问URI

  • Proxy-Authenticate 把代理服务器所要求的认证信息发送给客户端

  • Retry-After 多久之后再来访问

    一般配合两个来使用

    • 503 Service Unavailable
    • 3xx Redirect

  • Server 告知客户端服务器的名称

  • Vary 代理服务器检查可接收的自然语言

    • 代理服务器仅向源服务器请求Vary字段里的自然语言

  • WWW-Authenticate 是对 HTTP的访问认证

8 实体首部字段

  • Allow 服务端告诉客户端自己支持的方法有哪一些

  • Content-Encoding 服务端告知客户端采用的什么编码方式

    • gzip
    • compress
    • delate
    • identity

  • Content-Language 告知客户端，实体主体使用的自然语言

  • Content-Length 表明实体主体的大小

    • 而实体主体进行内容编码时，不能再使用该字段

  • Content-Location 表名实体主体对应资源的URI

    • 例如：首部字段可能

      Content-Location: https://www.cnmi.com

    • 实体的可能为

      Content-Location: https://www.cnmi.com/index.html

  • Content-MD5 作为检验使用，客户端会与服务端执行相同的MD5 算法，并与首部字段 Content-MD5 进行比较

    • 大体过程：
      • 首先，报文主体会进行MD5 算法
      • 其次，将算法得到的结果在进行Base64 编码
      • 最后，将编码的结果放在实体字段里面

  • Content-Range 发送其范围

    Content-Range:bytes 5001-10000/10000
    1. 总大小为1W字节
    2. 本次发送 5001-10000 这个范围里面的

  • Content-Type 表示实体主体的媒体类型以及字符集

    Content-Type: text/html; charset=UTF-8

  • Expires 服务端将告诉客户端该资源的失效的日期

    • 首部字段的 max-age 比实体的优先级高

  • Last-Modified 指明资源最后的修改日期

9 Cookie

两个字段

  • 服务端 Set-Cookie 响应客户端
  • 客户端 Cookie 客户端请求，发送给服务端

服务端

属性 解释
Name=Value 名称与值
expires=DATE 有效期
path=PATH 文件目录作为Cookie的适用对象
domain=域名
Secure 仅支持HTTPS请求时，发送Cookie
HttpOnly 避免 XSS 攻击，即跨脚本攻击

10 其他首部字段

  • 首部字段可以通过自行扩展实现
  • X-Frame-Options:DENY
    • HTTP 响应的首部，控制Frame标签的显示问题
    • 现在好像都没有这个标签了
  • X-XSS-Protection
    • HTTP 响应首部
    • 0 : 有效； 1 : 无效
  • DNT
    • HTTP 的请求首部
    • Do Not Track 拒绝个人信息被收集
    • 0 : 同意； 1 : 拒绝
  • P3P
    • HTTP 的响应首部
    • 保护隐私使用
