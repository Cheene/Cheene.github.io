HTTP协议简述

1 协议用在哪里？

客户端和服务端之间的通信

2 数据如何交换？

请求和响应

  • 请求：即由客户端发出
  • 响应：即由服务端发出

服务端不会主动建立连接，而是一直在监听客户端的连接请求

3 无状态协议 stateless

即协议不对发送和请求做持久化处理，如何能够对数据保持呢？

采用 cookie 的方式

4 请求，对方的地址是什么？

请求URI定位资源

GET https://www.cnmi.com HTTP/1.1

GET /index.html HTTP/1.1
Host:hacker.jp

获取HTTP服务器支持的HTTP种类

OPTIONS * HTTP/1.1

5 HTTP 有哪些方法呢？

GET 获取资源

  • 文本：返回本身
  • 类 CGI 程序：返回运算的结果

POST 传输实体主体

  • POST 目的不是来获取响应的主体内容

PUT 传输文件

  • 默认关闭，由于不带验证机制
  • 开启
    • 配合WEB应用程序的验证机制
    • 架构设计采用REST{ REpresentational State Transfer}标准

HEAD 获取报文首部

  • 只有首部，没有主体

DELETE 删除文件

OPTIONS 询问支持的方法

TRACE 追踪路径

  • 保存到达服务器的每个代理服务器
  • 设置首部字段 Max-Forwards

CONNECT 用隧道协议连接代理

6 持久连接

在不建立连接情况下

  • 建立TCP连接
    • SYN
    • SYN/ACK
    • ACK
  • 数据通信
    • HTTP 请求
    • HTTP 响应
  • 断开TCP连接
    • FIN
    • ACK
    • FIN
    • ACK

出现的问题是什么？

每次请求都要都需要建立连接，请求和响应，以及断开连接。

持久化

  • 客户端与服务端仅需要建立一次连接，同时就可以多次发起请求和响应

  • 支持持久化是客户端和服务端双方同时支持才可以

管线化

皆在可以同时发送请求而不需要等待每一次的响应

7 状态管理 Cookie

保存状态信息

过程

  • 服务端向客户端响应

    • Set-Cookie首部字段信息

    • <Set-Cookie:sid=1316511156;path=/;ex=xxxx>

      客户端再次请求时，自动携带Cookie信息

  • 服务器发现客户端的Cookie后

    • 对比服务器上的记录返回之前的状态信息

    • Cookie: sid=1316511156
