一 前言

1 网络分层机制

网络的分层，能够对各个部分进行封装。在进行修改时候，不需要全部替换而仅是需要替换部分。

各层之间通过接口访问，内部实现隐藏，优势在于每一层内部仅需要做好自己的事情。

2打包机制

对于TCP/IP协议

  • 自应用层到链路层是将数据层层打包，并添加对应的数据头。
  • 自数据链路层到应用层是层层解包的逆向过程。

3 三兄弟 DNS,HTTP,IP

  • DNS 域名转IP
  • HTTP WEB协议
  • IP 路由和多路选择

4 过程？

  1. URL 输入域名地址
  2. DNS： 域名地址转IP地址，通过DNS
  3. TCP 打包到IP
  4. RAP： 将 IP地址 转 MAC
  5. 数据到数据链路层
  6. 转换为电信号或其他信号

出现的问题是什么？

每次请求都要都需要建立连接，请求和响应，以及断开连接。

持久化

  • 客户端与服务端仅需要建立一次连接，同时就可以多次发起请求和响应

  • 支持持久化是客户端和服务端双方同时支持才可以

管线化

皆在可以同时发送请求而不需要等待每一次的响应

2 编码，提高传输的速率

速率，就是在单位的时间内能够发送更多的有效信息

通过编码来完成，而编码需要计算完成，需要消耗CPU资源。

两大主体差异

  • 报文(message)
    • 通信的基本单位，8位组字节流，通过HTTP通信传输
  • 实体(entity)
    • 作为请求或响应的有效载荷数据被传输

通常，报文主体等于实体主体。只有当传输中进行编码操作时，实体主体的内容发生变化，才导致它和报文主体产生差异

常用编码

  • gzip(GNU zip)
  • compress
  • deflate zlib
  • identity 不进行编码

分割发送的分块传输编码

让浏览器能够逐步显示页面，即在大容量页面传输时，进行数据的分割

多种数据的多部份对象集合

  • multipart/form-data
    • 首部字段需要通过Content-Type 来指定类型
  • multipart/byteranges
    • 206 部分内容 Partial Content

内容的范围请求

场景：即实现断点续传，在网络中断的情况下，如果网络能够修复，便继续从中断处恢复下载

  • 从 开始到3000 Range： bytes=-3000，5000-7000

内容协商 Content Negotiation

让服务端给客户端最合适的资源，比如中英文网站

  • Accept
  • Accept-Charset
  • Accept-Encoding
  • Accept-Language
  • Content-Language

协商的三个地方

  • 服务器协商

  • 透明协商

五 Web服务器

1 单台虚拟主机的多个域名

一个虚拟主机可以同时搭载多个WEB服务

在HTTP请求的时候，会将域名转换成DNS，那么返回的是IP地址

在同一个主机上，搭载多个WEB服务，而IP地址相同又该如何区分？

  • 此时必须在 Host 首部内完整指定主机名或域名的URI

2 数据转发三剑客

在客户端和服务端之间往往还有帮忙转发的其他应用程序

  • 代理 ,中间人
  • 网关，跨网传输
  • 隧道，加强安全

2.1 代理

任务 接收客户端的请求，并转发到源服务器

联动 可多级联动，每次转发也保存 Via 首部信息

两种代理

  • 缓存代理
    • 预先将资源的副本保存在代理服务器上
  • 透明代理
    • 不需要对报文做任何的加工而是透明的传输

2.2 网关

主要任务 将HTTP协议通信，转换到其他的协议通信，并且访问非HTTP服务器

  • 客户端到网关的链路可加密
  • 网关可以实现多级系统联动，比如购物结算时与信用卡系统联动

2.3 隧道

客户端与服务器进行安全的通信

隧道仅负责安全，不对数据做任何处理，即透明的

3 缓存在保存资源时

缓存是什么？

缓存是服务端资源的拷贝，经常在两个位置

  • 代理服务器
  • 本地浏览器

缓存的时效性

毕竟缓存是最后一次请求时数据的备份，总会有有效期

缓存服务器

即，缓存是保存在服务端，而不是本地

  • 客户端向服务端请求的时候，会经过缓存服务器，并拦截
  • 缓存服务器根据自己的缓存直接返回给客户端
  • 如果没有缓存或者缓存需要更新时
    1. 客户端 发送请求消息到达代理服务器
    2. 代理服务器 发送请求到达 资源服务器
    3. 资源服务器 响应 代理服务器 的请求
    4. 代理服务器 响应 客户端 的请求

客户端的缓存

即在浏览器种的临时文件，请求时，如果判定缓存未过期，网页会从本地存储里面读取，这样就会产生两种现象

  • 本来网速慢，但是长浏览的网页打开很快，甚至有时没有网也能打开
  • 网站明明更新了，为何打开还是不变，非要多刷新几次才可以

4 缓存代理的分类

  • 端到端首部 End-to-end Header

    • 可以通过代理而不再转发
    • 需要提供 Connection 首部字段
